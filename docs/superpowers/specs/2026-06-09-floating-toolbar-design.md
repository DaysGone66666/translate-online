# 网页悬浮工具条设计

## 目标

将现有网页右侧控件收缩为一个可上下拖动的小尺寸雷姆头像；仅悬停头像时显示翻译与设置按钮。

## 交互

- 雷姆头像固定吸附在网页右侧，不挤压网页内容。
- 默认仅显示半隐藏的 36 像素雷姆头像。
- 只有鼠标进入头像或键盘聚焦头像时，翻译与设置按钮才在头像上下方出现。
- 鼠标从头像移动到辅助按钮时维持展开，离开整个按钮组后自动隐藏。
- 用户按住雷姆头像上下移动；短点击头像打开翻译历史侧边栏。
- 拖动位置限制在当前视口内，并保存到 `chrome.storage.local`。
- 拖动头像后抑制本次点击，避免误开侧边栏。

## 按钮

- 翻译按钮：悬停头像后显示，复用当前页面翻译、取消翻译、显示或隐藏译文状态。
- 雷姆头像：唯一常驻入口，点击打开现有浏览器原生翻译历史侧边栏。
- 设置按钮：悬停头像后显示，打开扩展设置页。

## 架构

- `src/floating-toolbar-core.js`：提供位置限制和拖动阈值计算，保持可独立测试。
- `src/content-script.js`：渲染工具条、处理指针事件、调用现有翻译与扩展消息。
- `src/shared.js`：定义工具条位置存储键。

## 可访问性

- 三个按钮均提供明确的 `aria-label`。
- 键盘聚焦头像或辅助按钮时保持展开。
- `prefers-reduced-motion` 下禁用展开和移动动画。

## 测试

- 位置始终限制在视口内。
- 微小指针移动不会被判断为拖动。
- 内容脚本包含三按钮、悬停展开、指针拖动和位置持久化逻辑。
- 原有页面翻译按钮状态行为保持不变。
