# 站点术语库设计

## 目标

为 HLTV 等术语密集网站提供站点限定的固定翻译与术语约束，减少机械翻译、统一 CS 术语，并允许用户补充自定义术语。

## 翻译顺序

1. 根据当前网站、源语言和目标语言加载适用术语。
2. 文本与术语源词完全一致时，直接返回固定译文，不请求翻译服务。
3. 长句包含术语时：
   - DeepSeek 请求中加入本句命中的术语约束。
   - 免费引擎返回后仅对完整单词或完整短语进行校正。
4. 未匹配术语时沿用现有翻译流程。

## 术语来源

- 内置 HLTV/CS 术语库仅对 `hltv.org` 及其子域名生效。
- 自定义术语保存在 `chrome.storage.sync` 的 `custom_glossary_entries`。
- 自定义格式为每行 `源词 = 译文`，空行和以 `#` 开头的行忽略。
- 自定义术语应用于所有网站，并覆盖内置术语；首版同样限定为英文到简体中文。

## 安全边界

- 术语匹配不区分英文大小写，但必须匹配完整单词或完整短语。
- 队伍名、选手名、地图名等专名默认不替换。
- 仅在源语言为英语或自动检测结果为英语、目标语言为简体中文时使用首版术语库。
- 网络翻译缓存按主机名隔离；术语配置变化时清空缓存。

## 首版内置术语

包含 HLTV 高频界面短语与 CS 赛事表达，例如：

- `Player of the week` → `本周最佳选手`
- `Today's matches` → `今日比赛`
- `Today's news` → `今日新闻`
- `Complete ranking` → `完整排名`
- `Event calendar` → `赛事日历`
- `Overview` → `概览`
- `Matches` → `比赛`
- `Results` → `赛果`
- `Stats` → `数据统计`
- `live` → `直播`
- `stage` → `阶段`
- `playoffs` → `淘汰赛`
- `group stage` → `小组赛`
- `upper bracket` → `胜者组`
- `lower bracket` → `败者组`
- `best of 3` → `三局两胜`

## 测试

- 完整短语命中时不发起网络请求。
- 非 HLTV 网站不应用内置 HLTV 术语。
- 长句只提取实际命中的术语。
- 匹配完整单词，不能将 `Stats` 误匹配到更长标识符中。
- 自定义术语覆盖内置术语。
